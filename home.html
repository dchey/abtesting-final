A/B Testing

Objective

Perform an A/B test and statistical analysis on two websites to gain insights into their designs.

Making and Deploying Websites

In studio 5, we modified online cactus shopping websites into two versions:

  • A version: Uses more than one background and text colors.
  • B version: Has bold text for cactus prices.

    • Creating Hypothesis for Statistical Tests

    First, we have to create null and alternative hypotheses for two metrics:

    Time to completion:
  • Null hypothesis: Version A has faster time of completion because having more than one color helps showing the differences better at a glance.
  • Alternative hypothesis: Version B has faster time of completion than version A.
  • Reasonings behind alternative hypothesis: Part of the task is to make sure that the cart is over $150. Since version B highlights the prices with bold task, this could be more affective than using more than one colors.

    • Return rate:
  • Null hypothesis: Version A has more return rate than version B because different background colors help show different parts of the website.
  • Alternative hypothesis: Version B has more return rate than version A.
  • Reasonings behind alternative hypothesis: Version A uses only one hue(green) with different saturation and brightness, which might not help people see things more clearly, and background colors of checkout buttons are same for both versions.

    • Collecting Data and Applying Scripts

    Data Collection Process:

    I sent my link to my peer classmates during studio hours for data collection. They were randomly given one of the two versions, and had to fill at least $150 of cacti/succulents, go to the checkout page, and come back to the main page.

    Extracting and Transitioning Data:

  • I followed step 5 from studio 5 readme to extract and filter data from my website.
  • Then I turned the text file into csv file without any scientific notations.

    • Cleaning and Running Data:

  • In the csv file, I hand computed time to completion and counted return rate for both versions.
  • Then I passed these values to python file from studio 6 to compute p values, t score (time for completion), and chi^2 value (return rate).

    • Conclusions and Takeaways

    Conclusions from Calculations

    Time for completion:
  • t-score: 0.996312262747
  • p-value: 0.168641303251

    • Return rate:
  • chi^2 value: 0.405910609858
  • p-value: 0.524052450153

    • Conclusions:
  • Since p-values are greater than 0.05 for both time for completion and return rate, we fail to reject null hypothesis.
  • The high p-values for both metrics tell that results are not statistically significant, which tells that the tests did not show any significant difference between A and B.

    Takeaways:

  • The results show that changing or adding different colors, fonts, or styles does not significantly make the website more effective.
  • I collected less than 40 data for testing websites, which was not enough to get accurate results. Having more data could have made the results more accurate.
  • Types of data gathered should be important too; one of the datas for time of completion was a complete outlier, which tells that there is a high chance that it is not a good one for getting insight into the websites.
  • Conclusions of more complex designs (ex. website has checkout button located top right, user can see the total price after adding/removing stuff to the cart, etc.) could be more complex because you have to figure out how each design principle affects user experience.
